# Planfix API Client Usage Skill

## Overview
- Create a Claude Code skill at `skills/planfix-api-client/` that teaches an agent how to use
  this Planfix TypeScript API client to write new automation scripts and query/mutate Planfix data.
- **Problem it solves:** the client is autogenerated (18 `*Api` classes, 100+ endpoints) with
  conventions scattered across `AGENTS.md`, `docs/`, and ~20 example scripts. A new contributor
  (human or agent) has no single entry point. This skill is that entry point.
- **Integration:** the skill is documentation only — it references the real `src/config.ts`
  helper, the generated `*Api` classes, the `docs/*.md` references, and the `examples/` patterns.
  It ships one compile-checked reference example (`examples/skill-quickstart.ts`) so the documented
  patterns are guaranteed to match the actual generated API surface.

    `objectApi.getObjectById({ id: templateId })` then build a `Map<name, id>` from

  - Pagination: `examples/contacts-export.ts` — `offset`/`pageSize` loop, break on short/empty page.

## Testing Strategy
- **Unit tests:** the project has `tests/basic.test.ts` and `tests/integration.test.ts`. This skill
  ships no new library code, so no new unit tests are required — but `npm test` must still pass
  (no regressions) in the final verification task.
- **Compile-check (primary verification):** `examples/skill-quickstart.ts` typechecks via
  `npm run build`. Every `*Api` class name, method name, and request/response field cited in the
  reference docs must appear in the quickstart so tsc validates it. A doc claim with no
  corresponding compiled line is unverified and must be added to the quickstart or removed.
- **No e2e/UI tests:** this is a CLI/library project with no UI. The quickstart's run-guard must
  perform **only a harmless read** (or nothing) — never a live mutation — so accidentally running
  it is safe.

## Technical Details
- **Skill frontmatter:** YAML with `name: planfix-api-client` and a `description` rich in trigger
  phrases so the skill activates on Planfix-client tasks. Keep `SKILL.md` body short; push detail to
  `references/`.
- **Reference files:** `api-surface.md`, `querying.md`, `mutations.md`, `writing-scripts.md` under
  `skills/planfix-api-client/references/`.
- **Compile-check gate:** `examples/skill-quickstart.ts` must import from `../src/config` and
  `../src/generated` and exercise every API symbol cited in the docs. Its run-guard executes only a
  harmless read (or nothing); all mutation calls are constructed for typechecking but not sent.
- **Verification command:** `npm run build` (tsc). If `examples/` is outside the build's `include`,
  Task 1 pins an explicit `npx tsc --noEmit` invocation that covers the quickstart and uses it as
  the gate for every subsequent task.
- **Do not edit `src/generated`** — read-only reference per AGENTS.md.
